INTRODUCCION

Acontonuacion se analizara el caso de una empresa que tiene 25 empleados que pierden informacion constantemente, su sistemas es poco eficiente y lento. El proposito de esta guia es identificar las necesidades de la empresa  sin proponer
soluciones técnicas.

Identificación de necesidades explícitas

- El sistema funciona de forma lenta 
- Se pierde una gran cantidad de archivos 
- El sistema se cae de manera recurrente 


Identificación de necesidades implícitas

- El ritmo de trabajo es lento y poco productivo para trabajar diariamente
- No se puede trabajar de manera fluida y eficiente
- No existe una base fiable ya que los archivos se pierden
- No existe un progama confiable ya que este se cae y pierde datos constantemente

Clasificación de necesidades

-  El sistema funciona de forma lenta  / Operativa / Retraza el tiempo de las actividades diarias del proyecto
- Se pierde una gran cantidad de archivos / Tecnica / Afecta a la  obtencion, fiabilidad y retencion de los datos de la empresa
- El sistema se cae de manera recurrente  / Operativa / Hace menos fluido y eficiente el trabajo de los empleados
- El ritmo de trabajo es lento y poco productivo para trabajar diariamente / Operativa / Empeora el rendimineto de la empresa y sus resultados

- No se puede trabajar de manera fluida y eficiente / Operativa / Los empleados no pueden llegar a sus objetivos o completar metas de manera eficiente

- No existe una base fiable ya que los archivos se pierden / Tecnica/ No existe una conservacion de datos y compromete los archivos de la empresa

- No existe un progama confiable ya que este se cae y pierde datos constantemente / Técnica / No existe integridad alguna en los datos y confianza en la empresa

 Formular requerimientos iniciales

- El sistema debe permitir realizar las tareas sin problemas / Sistema lento / Funcional /
- El servicio debe mantenerse disponible durante todo el dia / Caídas del sistema / No funcional /
- La información debe conservarse sin perder datos / Archivos perdidos / No funcional /
- El sistema debe funcionar de manera continua y estable / Programa no confiable / No funcional /
- El sistema debe ser fluido y eficiente / Bajo rendimiento de trabajo / Funcional /
